# Skill 08: Adapt To New Tournament

## Use When

Someone wants to reuse the framework for another contest.

## Method

1. Run Skill 01 to map rules and payout.
2. Run Skill 02 to encode scoring.
3. Run Skill 03 to build truth inputs.
4. Run Skill 04 only for uncertain or high-leverage events.
5. Run Skill 05 to estimate field behavior.
6. Run Skill 06 to simulate leaderboard.
7. Run Skill 07 to choose a portfolio.
8. Document what is direct data, proxy data, model assumption, and private/local data.

## Minimal Public Example

Keep examples anonymous:

- `Team A`, `Team B`
- fake odds
- fake field hints
- no raw CSV scrape
- no private names

## Output

- runnable example
- data contract
- README note explaining objective
- tests proving scoring, probability normalization, field normalization, and reproducible simulation
