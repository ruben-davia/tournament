# Skill: Explain Recommendations

## Use When

Use this after strategy ranking to communicate the result to a human.

## Inputs Needed

- strategy summary
- selected picks
- probability diagnostics
- field diagnostics
- known risks

## Questions To Ask

- Is the user risk-seeking or risk-averse?
- Do they need a short answer or a full audit?
- Are any picks emotionally or practically unacceptable?

## Steps

1. State the recommended strategy.
2. Explain the key drivers.
3. Separate truth edge from field leverage.
4. Call out downside scenarios.
5. Give a concise action list.

## Output

A clear recommendation with rationale, risks, and alternatives.

## Common Mistakes

- Overselling precision.
- Hiding assumptions.
- Explaining every number instead of the decision.
